## Assumptions
- Implementer runs on Linux/macOS with Node.js >= 18 and `npm`; no real chain RPC endpoints are required.
- Real chain RPC endpoints are user-supplied via `.env`; missing values cause the chain to be skipped, not the server to crash.
- The bundled sanctions list is a small public sample to demonstrate structure, not a complete OFAC dataset.
- The MCP server uses the official `@modelcontextprotocol/sdk` over stdio with its default framing.
- All tool inputs and outputs are JSON serialisable; no filesystem or network access outside `.env`-configured RPCs and the bundled static data.
- The bubblewrap-safe environment allows `npm install`, `npx jest`, and `node src/server.js` to run.

## Implementation Notes
- **Tech stack:** Node.js >= 18, `@modelcontextprotocol/sdk` (latest stable v1.x), `ethers` v6, `dotenv`, `jest`. No external graph library — write a custom adjacency-list BFS in `src/graph/traversal.js`.
- **File touchpoints (key):**
  - `src/config/chains.js` — static chain registry (chainId, name, env-key, native symbol, explorer) for the seven required chains.
  - `src/config/bridges.js` — known bridge contract addresses per chain (Stargate, Across, Hop, LayerZero endpoints) with minimal ABI fragments for log decoding.
  - `src/utils/{errors,checksum,cache,rateLimit}.js` — typed errors, EIP-55 validation, TTL cache, serialised rate-limit gate.
  - `src/rpc/provider.js` + `src/rpc/fallback.js` — `getProvider(chain)` and `callWithFallback(fn, {retries=3, backoffMs=200})`.
  - `src/graph/traversal.js` (BFS with `MAX_GRAPH_DEPTH`, visited set) and `src/graph/clustering.js` (connected components + degree ranking).
  - `src/scoring/confidenceEngine.js` — declared weights summing to 1.0: `noncePattern 0.10`, `deploymentDna 0.15`, `bridgeHop 0.10`, `gasBehavior 0.10`, `eoaCluster 0.15`, `contractOverlap 0.15`, `temporalCorrelation 0.10`, `entropyScorer 0.05`, `sanctionProximity 0.10`. Tiers: `>= 0.70` High, `>= 0.45` Probable, else Speculative.
  - `src/heuristics/*.js` — 9 modules, all return canonical `{score, evidence, fired}` shape.
  - `src/data/sanctions.js` — sample list (≥8 addresses, header notes illustrative nature) for `sanctionProximity.js`.
  - `src/tools/*.js` — 11 adapters, each `{name, description, inputSchema, handler}`. `src/tools/index.js` exports a registry.
  - `src/server.js` — only file that reads `process.argv`; uses MCP SDK to register and connect over stdio.
- **Testing strategy:** All heuristics accept a `ctx` with `provider`, `cache`, `rateLimit`. Tests use `tests/fixtures/rpcMock.js` to inject mocked provider responses — no real network calls. Examples use an in-process `FakeMcpClient` that imports tool handlers directly, so `npm run example:*` works without an MCP runtime.
- **Sequencing (TDD-style, 2-5 min steps):**
  1. Foundation: `package.json`, `.env.example`, `src/config/{chains,bridges}.js`, `src/utils/{errors,checksum,cache,rateLimit}.js` + tests.
  2. RPC: `src/rpc/{provider,fallback}.js`, `tests/fixtures/rpcMock.js`, tests.
  3. Graph + scoring: `src/graph/*.js`, `src/scoring/confidenceEngine.js`, tests.
  4. Heuristics batch A (deterministic): `noncePattern`, `deploymentDna`, `entropyScorer`, `contractOverlap` + tests.
  5. Heuristics batch B (RPC-driven): `bridgeHopTracer`, `gasBehavior`, `eoaClusterGraph`, `temporalCorrelation`, `sanctionProximity` + `src/data/sanctions.js` + tests.
  6. Tools: 11 tool adapters + registry + tests.
  7. Server: `src/server.js` registers and connects over stdio.
  8. Examples: 3 workflow scripts + `npm run example:*` wiring.
  9. Docs: `README.md` (14 sections) + `docs/architecture.md` (6 topics).
  10. Final verification: `npm install && npm test && npm run example:basic && npm run example:trace && npm run example:sanctions`.
- **Risk controls:** Pin major versions of `@modelcontextprotocol/sdk` and `ethers`; isolate SDK usage in `src/server.js` and ethers usage in `src/rpc/`; have heuristics tolerate missing env URLs (`getProvider` returns `null`, caller skips that chain).
- **Most important verification commands:** `npm install`, `npm test`, `timeout 2 node src/server.js < /dev/null`, `npm run example:basic`.

- A complete OFAC sanctions dataset — only a documented sample is bundled.
- Authentication, multi-tenancy, or per-client rate limiting.
- Persistence to disk or a database — cache is in-memory only.
- Web UI, dashboard, or visualization renderer — the spec's "graph payload" is JSON only.
- CI configuration, Docker, or deployment manifests.
- LLM prompt templates or tool-calling wrappers beyond the standard MCP schema.
- Non-EVM chains (Solana, Bitcoin, etc.).
